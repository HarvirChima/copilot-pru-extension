GitHub Personal Access Token

Your PAT is stored locally in your browser and is only ever sent to api.github.com. It is never transmitted to any third-party server.

How to create a Personal Access Token

  1. Go to github.com/settings/tokens (or Settings → Developer settings → Personal access tokens).
  2. Click Generate new token (classic).
  3. Give the token a descriptive name, e.g. "Copilot PRU Tracker".
  4. Select the required scopes:
    • read:user – read user profile information
    • copilot – access Copilot usage data (if available for your account)
  5. Click Generate token and copy it here.

Note: PRU (Premium Request Usage) data is only available via the API for certain Copilot plans. If the "Test Connection" succeeds but the popup shows "No data", your plan may not yet expose per-user PRU statistics through the GitHub REST API.
